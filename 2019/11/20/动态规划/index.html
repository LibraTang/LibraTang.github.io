动态规划

发表于 2019-11-20 | 更新于 2019-11-29

记住之前的答案

步骤

  • 问题拆解，找到问题之间的具体联系
  • 状态定义
  • 递推方程推导
  • 实现

前两点是重点，前两个步骤顺利完成的话，递推方程和代码实现都很轻松了

状态定义其实是思考在解决一个问题的时候我们做了什么事情，得出了什么样的答案，对于这个问题，当前问题的答案就是当前的状态

题目实战

爬楼梯

最基础的动态规划问题

题目来源：LeetCode70

解析

  • 问题拆解

    我们到第 n 个楼梯可以通过第 n-1 个楼梯和第 n-2 个楼梯，因此第 n 个问题可以拆解成第 n-1 个问题和第 n-2 个问题，然后又可以继续拆解，直到第0个问题

  • 状态定义

    通过问题拆解，我们可以将第 i 个状态定义为：从起点到达第 i 个楼梯的路径总数

  • 递推方程

    由前两个步骤容易得出递推方程：dp[i] = dp[i-1] + dp[i-2]

  • 实现

    我们需要一个初始值方便我们的计算，而初始位置不需要移动，所以dp[0] = 0，dp[1] = 1，dp[2] = 2，通过这几个初始值可以进行递推

三角形最小路径和

LeetCode120

步骤

  • 问题拆解

    对于这道题问题是求出最小路径和，关键是路径，和爬楼梯类似，[i][j]位置的元素肯定经过[i-1][j]或[i-1][j-1]，因此经过这个元素的路径可以拆解成上面的一个或两个元素的路径和得到

  • 状态定义

    这道思考有两种方式：一种是从上到下，一种是从下到上。如果用从上到下的方法，会发现在考虑下面元素的时候，起始元素的路径只会从[i-1][j]获得，而最后一个元素的路径只会从[i-1][j-1]获得，这样不方便实现，但考虑从下到上就不会有这个问题。状态的定义就变成：最后一行元素到当前元素的最小路径和

  • 递推方程

    dp[i][j] = Math.min(dp[i+1][j], dp[i+1][j+1]) + triangle[i][j]

  • 实现

    初始化时将最后一行的元素填入dp数组中，然后递推即可

不同路径

不同路径的进阶版，考虑了障碍物

LeetCode63

步骤

  • 问题拆解

    矩阵类动态规划需要关注当前位置和其相邻位置的关系，对于某一个位置来说，经过它的路径只能是从它上面或左边过来，因此要知道到达当前位置的不同路径，我们需要知道到达其上方和左方的不同路径。如果当前位置是障碍物的话，dp数组中当前位置的记录就是0,表示没有任何一条路径能到达这里。

  • 状态定义

    从起点到达该位置的不同路径数目

  • 递推方程

    dp[i][j] = dp[i-1][j] + dp[i][j-1]

  • 实现

    考虑dp数组的初始化问题，对于上边界和左边界的位置，因为它们只有一个方向能过来，需要单独考虑，不同路径个数就是1

最大正方形

LeetCode221

步骤

  • 问题拆解

    假设我们当前位置是正方形的右下角的点，然后我们要看之前遍历过的点有没有能和当前点组成正方形的，无非是看上方、左方和左上方，也就是从这个点向这三个方向延伸，具体延伸的距离和这三个点的状态有关

  • 状态定义

    由于我们考虑的是正方形右下角顶点，因此状态可以定义为：当前点为正方形右下角顶点时，正方形的最大边长

  • 递推方程

    我们需要取三个方向状态的最小值才能确保我们延伸的是全为1的正方形，因此递推方程为：dp[i][j] = Math.min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1

  • 实现

    实现时我们需要考虑两种情况，当前位置是 ’1‘还是’0‘，如果是’0‘的话当前状态就是0；如果是’1‘的话，我们也要考虑当前位置，如果是第1行或第1列的元素，该位置无法同时向三个方向延伸，状态为1即可，其他情况则按之前得出的递推方程求解

最长上升子序列

补充：子数组必须是数组中的一个连续区间，而子序列没有这样的要求，只需要保证子序列中的元素的顺序和原数组中元素的顺序一致即可

Leetcode300

步骤

  • 问题拆解

    子序列虽然不是数组中连续的区间，但依然有起点和终点。如果我们确定了终点的位置，然后再看前面 i-1 个位置中，哪一个位置可以和当前位置拼接在一起，这样就可以把问题拆解成思考之前的 i-1 个问题

  • 状态定义

    我们的状态可以定义为：以位置i结尾的子序列的最大长度

  • 递推方程

    dp[i] = Math.max(dp[j], ... dp[k]) + 1 (j < i, k < i)

  • 实现

    实现的时候，我们需要考虑数组的初始化，因为对于每一个位置，它本身就是一个序列，因此所有位置的状态都初始化为1

打家劫舍II

这道题特殊的地方就在于最后一个房子和第一个房子之间产生了联系，有个小技巧就是可以线性考虑[0, n-2]和[1, n-1]，然后求二者的最大值

Leetcode213

步骤

  • 问题拆解

    我们可以思考第 i 个房子是否应该抢，如果抢，那么第 i-1 个房子就不能抢，我们只能考虑抢第 i-2 个房子；如果不抢，那么就可以抢第 i-1 个房子，这样第 i 个房子和第 i-1 和第 i-2 个房子就联系上了

  • 状态定义

    我们将状态定义为：抢到第 i 个房子可以获得的最大值

  • 递推方程

    dp[i] = Math.max(dp[i-2] + nums[i], dp[i-1])

  • 实现

    因为第 i 个位置和前面的两个位置都有关，这个时候我们可以把状态多开一格，dp[0]表示一个房子都不抢的状态，dp[1]就是最开始的房子获得的最大价值
